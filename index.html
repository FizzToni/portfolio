FHIR - Health-App
Java, Kotlin, REST, MongoDB, FHIR
In the mobile computing class we were challenged to design a Health App using the FHIR Framework. We were free to use any tech-stack we wanted. So I used the one I was most familiar with, and the one that seemed most comfortable for me. I derived my Java backend-server from HAPI-FHIR and created my App with Kotlin in Android Studio. The two communicated with JSON over a simple REST api. Since I only needed a simple server to store the data I ended up using MongoDB for its nice interface. Users could set up their medical information to have it at hand in case of an emergency (ICE) and as a planner to have important appointments saved and contact information to various doctors or persons.
Arduino Home Automation
Arduino, C++
Many stand before the same problem at some point. Having a beloved basil-plant that now hase to face 2 weeks of drought because vacations are up ahead, and there is just no way, it fits in our bags. This, (and some not so dry walls) got me into programming my own micro-controllers. I built a mesh with some sensors at various places, so i can monitor everything on one screen. Since the wifi-module does not reach as far as I would like i ended up placing multiple arduinos-connected to a mesh so I can retrieve data even from the one in my bathroom. I monitor the humidity behind couches and wardrobes and update a list of outdoor and indoor temperature. One plant's soil is being monitored and watered by an automatic pump... if there is water in the tank. The plant still lives.
